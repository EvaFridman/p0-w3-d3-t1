# p0-w3-d3-t1 · Разбор лог-ленты

> Тип: циклы · Репозиторий: один отдельный

> Подключить файл, отсмотреть в браузере.

## Задача

Сервис пишет лог строками, и нужно быстро понять, что в нём происходило: сколько было ошибок, сколько успехов, где первая ошибка. Это типичная работа циклом по списку записей с управлением ходом через `continue` и `break`.

```
массив лог-записей → пройти циклом for...of
  'INFO ok' · 'ERROR timeout' · 'INFO ok' · 'WARN'
          │  continue — пропускаем WARN
          │  считаем INFO и ERROR отдельно
  итог: {info: 2, error: 1}  + первая ошибка (break)
```

## Что нужно сделать

В `logscan.js` взять готовый массив логов (ниже). Пройти его через `for...of` и посчитать, сколько записей уровня `INFO` и `ERROR`; строки `WARN` пропускать через `continue`. Отдельно — найти и вывести первую `ERROR`, остановив перебор через `break`.

Входные данные:

```js
const logs = [
  "INFO server started",
  "INFO request /api/users",
  "WARN slow query 820ms",
  "ERROR db connection refused",
  "INFO request /api/orders",
  "ERROR timeout on /api/pay",
  "WARN retry queue growing",
];
```

Ожидаемый результат (примерно):

```
INFO: 3, ERROR: 2
Первая ошибка: "ERROR db connection refused"
```

## Критерии приёмки

* Массив обрабатывается циклом `for...of`, отдельно посчитаны `INFO` и `ERROR`.
* `WARN` пропускаются через `continue`, поиск первой ошибки останавливается через `break`.
* Понятно, чем `for...of` отличается от перебора по индексу и когда уместнее `break` vs `continue`.
